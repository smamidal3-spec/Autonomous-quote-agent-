Learning Types

Supervised vs Unsupervised Learning

✅ Supervised
  • We train on labelled data: past quotes tagged HIGH / MEDIUM / LOW risk
  • Model learns: age + accidents + vehicle cost → risk tier
  • Used for: Risk Profiler, Conversion Predictor, Premium Advisor
  • Human verified outcomes improve model accuracy over time
  • Accuracy validated on 20% held-out test set before any deployment
🔎 Unsupervised
  • No labels needed — model finds hidden structure in data automatically
  • Isolation Forest detects unusual patterns that deviate from norm
  • KMeans clusters customers into similar behavioural groups
  • Used for: Fraud Detection, Personalisation Agent
  • Continuously adapts as customer behaviours evolve and shift

Agent 1 of 6

Risk Profiler

Inputs
  • Driver Age
  • Driving Experience
  • Prior Accidents
  • Citation Count
  • Vehicle Cost Range
  • Coverage Type
Logic
  • → Random Forest model trained on 146K clean policy records
  • → Heuristic overrides applied for extreme young/old drivers
  • → SHAP values pinpoint top 3 risk drivers per quote
  • → Outputs: LOW / MEDIUM / HIGH risk tier + numeric score
  • → MEDIUM+ tier auto-elevates to underwriter review queue
Outputs
  • ✦ risk_score (0–100)
  • ✦ risk_tier: LOW / MED / HIGH
  • ✦ top 3 SHAP drivers
  • ✦ heuristic_override flag
  • MEDIUM or higher tier automatically triggers underwriter review queue — no manual triage needed
  • SHAP explanations are stored with every decision for regulatory audit trail
Agent 2 of 6

Fraud Detection

Inputs
  • Driver Age vs Exp delta
  • Vehicle cost vs salary ratio
  • Accident velocity
  • Premium vs risk gap
  • Re-quote flag
Logic
  • → Validation rules fire first (18+ age, exp ≤ age-16)
  • → Isolation Forest anomaly model scores 12 engineered features
  • → Rule engine flags underwriting anomalies (salary ≠ vehicle)
  • → fraud_flag set ONLY if anomaly score ≥ 0.5 threshold
  • → rule_flags are SEPARATE — trigger escalation, not fraud label
Outputs
  • ✦ fraud_score (0.0–1.0)
  • ✦ fraud_flag: True / False
  • ✦ rule_flags: [list]
  • ✦ fraud_reason_codes
  • Fraud flag and rule flags are deliberately separated — different impact on routing and fairness
  • Rule flags trigger escalation without a fraud label — maintains regulatory fair treatment
Agent 3 of 6

Conversion Predictor

Inputs
  • All 18 quote features
  • Risk tier (from Agent 1)
  • Coverage choice
  • Annual miles
  • Salary range
Logic
  • → XGBoost classifier trained on labelled bind/no-bind outcomes
  • → Removes Policy_Bind_DT date column (leakage eliminated)
  • → Outputs bind probability 0.0–1.0 → banded to HIGH/MED/LOW
  • → SHAP values surface top 3 conversion drivers per quote
  • → LOW conversion triggers discount recommendation in Agent 5
Outputs
  • ✦ conversion_probability
  • ✦ conversion_band: HIGH/MED/LOW
  • ✦ top_conversion_drivers (SHAP)
  • ✦ discount_recommended flag
  • Policy_Bind_DT date column fully removed — eliminates 100% of identified model leakage
  • AUC 0.82 on held-out test set — model generalises to unseen data correctly
Agent 4 of 6

Personalisation Agent

Inputs
  • Salary range
  • Vehicle cost
  • Annual miles
  • Coverage preference
  • Risk tier
Logic
  • → KMeans clustering on 5 behavioural dimensions
  • → Customer mapped to nearest centroid: Budget / Standard / Premium / Fleet / Senior
  • → Add-on recommendations generated from cluster profile
  • → Expensive vehicles → zero-depreciation + engine protection
  • → Young drivers → personal accident cover auto-included
Outputs
  • ✦ customer_segment
  • ✦ recommended_plan
  • ✦ add_ons: [list]
  • ✦ segment_confidence
  • Five segments — Budget · Standard · Premium · Fleet · Senior — each mapped to a distinct plan tier
  • Add-on rules fire automatically per segment profile — no manual configuration needed
Agent 5 of 6

Premium Advisor

Inputs
  • Quoted premium (from form)
  • Risk score (Agent 1)
  • Conversion probability (Agent 3)
  • Salary range
Logic
  • → Calculates expected premium band from risk × salary alignment
  • → Compares quoted vs expected — flags over/under-pricing
  • → If salary ≤ ₹25 Lakh AND quoted premium > 2× expected → issue
  • → Generates recommended premium min/max range
  • → Borderline cases include discount coupon recommendation
Outputs
  • ✦ premium_issue: True / False
  • ✦ recommended_range [min, max]
  • ✦ recommendation_reason
  • ✦ discount_applicable flag
  • Prevents under-pricing of high-risk profiles — protects the loss ratio at source
  • Prevents over-pricing of low-risk customers — directly improves conversion rate
Agent 6 of 6

Decision Router

Inputs
  • risk_tier
  • fraud_flag
  • rule_flags
  • conversion_band
  • premium_issue
Logic
  • → Deterministic rule engine — fully auditable, no black box
  • → AUTO-APPROVE: LOW risk + HIGH conversion + no flags
  • → ESCALATE: HIGH risk OR fraud_flag OR any rule_flag present
  • → FOLLOW-UP: MEDIUM risk OR LOW conversion OR premium issue
  • → Confidence score (0–1) reflects routing certainty
  • → Detailed explanation included for every decision path
Outputs
  • ✦ decision: AUTO/FOLLOW/ESCALATE
  • ✦ confidence_score (0–1)
  • ✦ decision_explanation
  • ✦ escalation_required flag
  • Fully deterministic — every routing path is auditable, no black-box routing
  • Written explanation attached to every decision for IRDAI regulatory compliance
