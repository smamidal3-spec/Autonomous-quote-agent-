Learning Types

Supervised vs Unsupervised Learning

Supervised

  • Trained on labelled outcomes — past quotes tagged HIGH / MED / LOW
  • Model learns: age + accidents + cost → risk tier
  • Used for Risk Profiler, Conversion Predictor, Premium Advisor
  • Human-verified outcomes improve accuracy over time
  • Accuracy validated on 20% held-out test set before deployment

Unsupervised

  • No labels needed — discovers hidden patterns automatically
  • Isolation Forest finds anomalies that deviate from the norm
  • KMeans groups customers into 5 behavioural segments
  • Used for Fraud Detection and Personalisation Agent
  • Continuously adapts as customer behaviour shifts

Governance

Monitoring, Governance & Retraining

Drift Detection
PSI tests run weekly on all feature distributions. Alert triggered if score exceeds 0.2 threshold.
Retraining Cadence
Models retrained every 90 days on latest labelled underwriter outcomes. Accuracy gate enforced.
Audit Logs
Every decision logged with inputs, agent outputs, final routing, confidence score and timestamp.
Compliance
IRDAI-aligned explainability. Every rejection includes a written reason code. SHAP stored per case.
  • No model deployed without 90%+ accuracy gate on held-out validation data
  • Feedback loop — underwriter decisions feed directly into next training cycle
